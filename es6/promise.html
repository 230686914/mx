    # Promise

    例：假设小明参加考试，及格了奖励100块，不及格惩罚100块

    // 上述例子对应到ajax则为：
$.ajax({
  url: '参加考试',
  success: (res) => {
    if(res >= 60){
      console.log('奖励100块')
    }else {
      console.log('惩罚100块')
    }
  }
})

    例：与此同时，小红也参加考试，及格了奖励200块，不及格惩罚200块

    // 上述例子对应到ajax则为：
$.ajax({

  success: (res) => {

    观察上面2个例子，会发现ajax请求有高度重合的部分，除了处理结果，其他都为高度重合代码。

    // 封装ajax重合部分代码，把不一样的地方抽离出来当做一个参数传递进去
function test(resolve, reject){
  $.ajax({
    url: '参加考试',
    success: (res) => {
      if(res >= 60){
        resolve();
      }else {
        reject();
      }
    }
  })
}

// 小明同学
test(() => {
  console.log('奖励100块')
}, () => {
  console.log('惩罚100块')
})

// 小红同学
test(() => {
  console.log('奖励200块')
}, () => {
  console.log('惩罚200块')
})

    上述封装代码完成，但是有的地方不太严谨，因为封装的方法不看内部代码的话，参数代表什么意思，不能明白，而且代码没有注释的时候，代码阅读起来很难。
     我们在开发的时候最讨厌的2件事：1、别人代码不写注释！ 2、自己讨厌写注释！

    基于上述情况，有人提出了 Promise 的概念，Promise 不是 es6 最早实现的，jq 里面最早有实现 Promise 方法 => $.Promise

    // 上述ajax封装，改为 Promise 
// 异步需求都可以用 Promise 来代替
// Promise 会返回一个对象,new Promise() 为一个构造函数，构造函数绑一个对象
var promise = new Promise((resolve, reject) => {
    $.ajax({
      url: '参加考试',
      success: (res) => {
        if(res >= 60){
          resolve();
        }else {
          reject();
        }
      }
    })
})

// 上述代码阅读起来会很方便，Promise有2个参数，一个为 成功的结果 - resolve, 一个为 失败的结果 - reject
// Promise通常配合then方法来链式的使用
// then方法里面第一个回调函数表示成功状态，也就是resolve通过.then调用
// 第二个是失败状态-reject通过.Cath调用
// 如果默认写一个参数的话，默认resolve
promise.then(res => {
  console.log('奖励100块')
}).catch(e => {
    console.log('惩罚100块')
})

    如果函数涉及到多个回调函数并且是异步的，可以考虑使用 Promise 来优化。
